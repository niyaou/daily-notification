# 审核清单（固定顺序）

## 1. 输入完整性
- [ ] PR/MR 元信息完整
- [ ] commit 列表完整
- [ ] diff 可读
- [ ] 现有文档可读

## 2. 四大检查项
- [ ] bug_check
- [ ] requirement_fit
- [ ] principle_conflict
- [ ] cross_impact

## 3. 文档一致性
- [ ] 触发条件命中时已更新需求或 features 文档
- [ ] 模块变更时已更新模块图
- [ ] 流程变更时已更新时序图
- [ ] 架构变更时已更新架构文档和 ADR

## 4. 输出格式
- [ ] Commit 级记录在前
- [ ] PR/MR 汇总在后
- [ ] 字段完整且枚举值合法
- [ ] `UNKNOWN` 项进入 `open_questions`
